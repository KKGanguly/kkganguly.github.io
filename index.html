I'm a first-year Ph.D. student in Computer Science at NC State, advised by Dr. Tim Menzies. My research focuses on making machine learning and optimization methods more practical and efficient for software engineering.

I work on problems at the intersection of automated testing, program analysis, and empirical software engineering—always asking how we can do better with less. Research

Core theme: Software engineering tasks are often easier than we think. My research shows that with the right insights—learning from real crashes, understanding problem structure, or using simpler methods—we can achieve better results with less effort.

Want to find bugs more effectively?

Coverage alone isn't enough. By learning from actual crash patterns, we can guide fuzzers to discover vulnerabilities faster.

Data-Centric Fuzzing for JavaScript Engines

with Tim Menzies | 2025

We developed a machine learning-guided fuzzing framework that learns from crash patterns to discover vulnerabilities in JavaScript engines. Our approach combines feature extraction from real crashes with SHAP-based mutation strategies, achieving 85%+ precision with minimal false alarms.

By integrating this model into the Fuzzilli fuzzer, we discovered a previously unknown V8 crash—demonstrating that data-centric approaches complement traditional coverage-based fuzzing.

Want to optimize faster with simpler methods?

Most SE optimization problems collapse to tiny search spaces. Simple methods can match complex algorithms with 10x fewer evaluations.

Efficient Multi-Objective Optimization for Software Engineering

with Tim Menzies and collaborators | 2024-2025

We discovered that multi-objective optimization problems in software engineering collapse to just 1-2% of their theoretical search space (the "BINGO effect"). This means simple optimizers can reach >90% of optimal performance with ≤50 evaluations, drastically reducing computational cost.

We also created MOOT, a standardized benchmark of 120+ real-world optimization tasks to enable reproducible research and fair comparison of methods.

Want agents to adapt efficiently without central control?

In today's world of agentic AI and microservices, systems need to self-adapt without central control. Reinforcement learning enables efficient coordination with minimal overhead.

Decentralized Self-Adaptation with Partial Knowledge

M.S. Thesis, University of Dhaka | 2017-2018

Developed a novel decentralized self-adaptation mechanism using Reinforcement Learning and Interaction-Driven Markov Games (IDMG). The key innovation: agents learn optimal decisions by coordinating with only a single peer, rather than requiring global knowledge.

The approach restricts coordination only to states where global goals are violated, allowing independent action otherwise. This achieved 33% higher rewards than joint Q-learning while reducing coordination overhead by >50%—critical for modern agentic systems and microservices where scalability matters.

Want better automated program repair?

Better patch ranking means fewer false positives. Combining syntactic and semantic signals improves repair quality.

Similarity-Based Patch Prioritization for Automated Repair

Analyzed how syntactic and semantic similarities affect patch prioritization in automated program repair. Showed that combining both measures significantly improves patch ranking quality across multiple benchmarks.
