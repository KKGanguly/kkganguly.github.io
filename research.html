Data-Centric Fuzzing for JavaScript Engines

2025

with Tim Menzies

We developed a machine learning-guided fuzzing framework that learns from crash patterns to discover new vulnerabilities in JavaScript engines. Traditional fuzzing relies on code coverage to guide test generation, but coverage-based approaches often miss semantic bugs that don't manifest in coverage metrics.

Our approach combines feature extraction from real crashes with SHAP-based mutation strategies. We built an iterative pipeline that derives 150+ static and runtime features from V8 crash-triggering inputs, then trained an XGBoost model on 160 reproduced crashes and 8,500+ benign tests. The model achieves 85%+ precision with less than 1% false alarms.

By integrating this model into the Fuzzilli fuzzer with SHAP-guided, feature-driven mutation, we discovered a previously unknown V8 crash. This demonstrates that data-centric approaches can complement traditional coverage-based fuzzing.

BINGO: Simple Optimizers for Software Engineering

2025

with Tim Menzies

We investigated why multi-objective optimization in software engineering often works better than theory predicts. Our key finding is the "BINGO effect": SE optimization problems collapse to just 1-2% of their theoretical search space.

This discovery has practical implications. Since SE problems have much smaller effective search spaces than their theoretical bounds suggest, simple optimizers like random search or grid search can reach >90% of optimal performance with just ≤50 evaluations. This is an order of magnitude fewer evaluations than complex evolutionary algorithms typically require.

The BINGO effect challenges conventional wisdom about the need for sophisticated optimization in software engineering. For many practical tasks—software configuration, cloud tuning, hyperparameter optimization—simple methods suffice if you understand the problem structure.

MOOT: Multi-Objective Optimization Tasks Repository

2025

with Tim Menzies, Tao Chen, Yulong Ye, and others

A major challenge in optimization research is the lack of standardized, real-world benchmarks. Researchers often use toy problems or synthetic data, making it difficult to assess whether methods work on practical software engineering tasks.

We created MOOT (Multi-Objective Optimization Tasks), a repository of 120+ real-world optimization tasks spanning software configuration, cloud tuning, and process modeling. Each task includes the problem specification, search space definition, and evaluation metrics.

MOOT enables reproducible research and fair comparison of optimization methods. It also revealed the BINGO effect across multiple domains, suggesting that the collapse of search spaces is a general property of SE optimization problems rather than an artifact of specific datasets.

Decentralized Self-Adaptation in Multi-Agent Systems

2017-2018

M.S. Thesis, University of Dhaka

Developed a multi-agent Q-learning framework using Interaction-Driven Markov Games (IDMG) for decentralized self-adaptive systems. The framework achieved 33% higher reward than joint Q-learning baselines while improving scalability by reducing coordination overhead by >50% through a dynamic state-identification mechanism.
